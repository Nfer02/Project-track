## Vision

SaaS web app de gestión financiera de proyectos para trabajadores autónomos. Permite registrar obras/proyectos, asociar facturas de gastos e ingresos, y visualizar en un dashboard el estado financiero en tiempo real: presupuesto vs. gasto real, ingresos cobrados, desviaciones y rentabilidad.

**KPIs de éxito:**
- Registrar una factura (foto → confirmación) en < 30 segundos via OCR
- Dashboard con datos actualizados sin recargar
- Conversión Free → Pro > 15% en los primeros 30 días

## Data Model

### Entidades

- **User** — cuenta de usuario (sync desde Supabase Auth)
- **Workspace** — unidad de suscripción (la empresa/autónomo). Plan FREE o PRO.
- **WorkspaceMember** — relación User ↔ Workspace con rol OWNER / EDITOR / VIEWER
- **Project** — obra o proyecto con presupuesto, cliente, fechas y status (ACTIVE / COMPLETED / PAUSED / ARCHIVED)
- **Invoice** — factura de tipo EXPENSE (gasto) o INCOME (ingreso), con campos: número, contraparte, base, IVA, total, fechas, status (PENDING / PAID / OVERDUE / CANCELLED), archivo adjunto, flag OCR

### Relaciones

- Workspace 1:N Project
- Workspace 1:N Invoice
- Project 1:N Invoice
- User N:M Workspace (via WorkspaceMember)

### Schema Prisma
El schema completo está definido en `prisma/schema.prisma` con todos los modelos, enums e índices tal como aparece en el blueprint (Sección 4).

## API

| Método | Ruta | Auth |
|--------|------|------|
| POST | /api/workspaces | Sí |
| GET/POST | /api/projects | Member / Editor+ |
| GET/PUT/DELETE | /api/projects/[id] | Member / Editor+ / Owner |
| GET | /api/projects/[id]/metrics | Member |
| GET/POST | /api/invoices | Member / Editor+ |
| PUT/DELETE | /api/invoices/[id] | Editor+ |
| POST | /api/invoices/scan | Editor+ — OCR con Claude API |
| GET/POST/DELETE | /api/workspaces/[id]/members | Owner |
| POST | /api/checkout | Owner — Stripe Checkout |
| POST | /api/webhooks/stripe | Sin auth (signature verification) |
| POST | /api/webhooks/supabase | Sin auth (secret header) |

### Endpoint crítico: POST /api/invoices/scan

Recibe `multipart/form-data` con un archivo (imagen o PDF).
- Imagen → base64 → Claude API vision
- PDF → soporte nativo de documentos de Claude API
- Prompt estructurado para extraer: número, contraparte, NIF, base, IVA, total, fecha, moneda
- Responde con campos extraídos + `confidence` (0–1)
- No crea la factura; solo extrae datos para pre-llenar el formulario

### Arquitectura de componentes clave

**ProjectDetail:** BudgetProgress (anillo SVG/recharts) + FinancialSummary (3 cards: ingresos/gastos/beneficio) + DeviationAlert (banner si desviación > 10%) + InvoiceList (últimas 5)

**NewInvoice:** InvoiceUploader (drag & drop + cámara mobile) → OcrPreviewModal (muestra extracción, permite editar) → InvoiceForm (pre-llenado)

### State Management

- Server Components para páginas de solo lectura (dashboard, listas)
- Client Components para formularios, uploaders, modales
- Server Actions para mutations + `revalidatePath`
- Sin Zustand ni React Query en MVP

## Auth & Roles

1. Registro en Supabase Auth → webhook sincroniza a tabla `users`
2. Onboarding crea Workspace + WorkspaceMember (OWNER)
3. Middleware Next.js protege `/(app)/*`, redirige a `/sign-in` si no hay sesión

| Rol | Permisos |
|-----|----------|
| OWNER | Todo + billing + gestión de miembros |
| EDITOR | Crear/editar proyectos y facturas |
| VIEWER | Solo lectura |

---

## Monetización

**Plan FREE:** 1 proyecto activo, máximo 20 facturas/mes
**Plan PRO:** Ilimitado — suscripción mensual via Stripe

Límites verificados en Server Actions antes de crear. Stripe webhooks actualizan `workspace.plan` y `subscription_status`.
